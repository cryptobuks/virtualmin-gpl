Créer un sous-serveur virtuel
Ce formulaire vous permet de créer un nouveau sous-serveur virtuel, qui est détenue par une connexion d'administration existant. Un sous-serveur partage également quota de disque, limites de bande passante et les limites sur les utilisateurs, les alias et les autres objets avec son parent.

Lorsque vous créez le serveur, vous devez entrer au moins le nom de domaine et une description. Si plusieurs modèles de serveur sont définies sur le système, vous pouvez sélectionner un dans le menu - modèles déterminent différents paramètres par défaut pour le nouveau serveur, tels que la configuration du site, les enregistrements DNS et ainsi de suite.

Dans la section Options avancées vous pouvez sélectionner un nom de base de données initiale différente, plutôt que d'utiliser la valeur par défaut qui est dérivé du nom de domaine. Les noms d'utilisateur et de groupe ne peuvent pas être modifiés, car ils sont toujours héritées du serveur parent.

Pour sélectionner les fonctions qui sont activées pour le serveur, utilisez les cases à cocher dans la deuxième partie de la section Options avancées. Certaines fonctionnalités dépendent des autres et ne peuvent pas être sélectionnés sur leur propre - par exemple, Webalizer (rapports sur les statistiques du site) peut fonctionner si vous avez réellement un site Web.

Dans la partie inférieure de la section Options avancées vous pouvez choisir d'avoir tout le trafic Web à ce nouveau site transmis à une autre URL, et d'avoir tous les messages transmis. Vous pouvez également choisir d'avoir le nouveau serveur virtuel utilise ses propres adresses IP privées, qui est nécessaire pour SSL et hébergement FTP anonyme.
